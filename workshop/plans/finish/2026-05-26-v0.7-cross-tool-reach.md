## Task 2: Cursor MDC frontmatter in scar/playbook templates

Update `skills/workshop-workflow/templates.md` — scar and playbook templates gain optional Cursor MDC frontmatter fields. The required field set (per Task 2 of v0.6) is `when_to_read` + `applies_to` + `last_updated`; v0.7 adds optional MDC bridge fields:

```yaml

alwaysApply: false  # optional; Cursor MDC always-load flag (default false)
```

Document that these fields are OPTIONAL and exist specifically to let Cursor / Continue / etc. scope-load scars/playbooks by file path without the SessionStart hook.

## Task 5: README "Why not just AGENTS.md?" section

Add new section to both `README.md` and `README.zh.md` (after the main feature description, before "Roadmap"). Positions workshop as the operating protocol on top of AGENTS.md the wire format. Key points:
- AGENTS.md is the cross-tool standard for project instructions (Linux Foundation, ~60k repos)
- AGENTS.md is static prose; workshop is lifecycle state machine + write gate + scar log + critique disposition + multi-session continuity
- workshop EMITS into AGENTS.md (via `/workshop:emit-agents-md`) — it doesn't replace AGENTS.md
- If you only need "static project rules for an AI", AGENTS.md alone is enough. If you need session-to-session continuity / lifecycle / scars, workshop fits on top.

Length: ~150 words EN + ZH mirror.
